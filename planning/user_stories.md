# User Stories

Built around the real people who will use *Codefolio*—the following user stories highlight the roles, goals, and motivations that guide the platform’s design. Each story helps ensure that every feature we build supports genuine user needs, while fostering a supportive, collaborative community of developers.

## User Roles

👤 Viewer (not logged in)

🎓 Student

💻 Junior Developer


## User Stories

1. As a junior developer, I want to connect with others to collaborate on larger projects, including open‑source, so that I can build a stronger portfolio.
1. As a viewer, I want to filter projects by category or tag so that I can discover projects in areas like web development, data science, mobile development, etc.
1. As a student, I want to showcase my skills, interests, and tech stack so that recruiters can quickly understand my strengths.
1. As a viewer, I want to like and comment on projects to show my appreciation, ask questions, or give feedback to the creator.
1. As a game developer, I want to find backend developers interested in creative projects so that I can build a well‑rounded team.
1. As a viewer, I want to browse a feed of uploaded projects so I can discover interesting work from other users.
1. As a junior developer, I want to get UI/UX feedback on my portfolio so that I can make it more professional.
1. As a viewer, I want to save or bookmark projects so I can come back to them later.
1. As a student, I want to find other student developers to team up with for hackathons so that I can gain experience and build my network.
1. As a viewer, I want to click into a project’s detail page so that I can see more information, the GIF walkthrough, and the GitHub link.
1. As a junior developer, I want to challenge myself with new projects so that I can continue improving my skills.
1. As a viewer, I want to search for projects by title or keyword so I can quickly find projects related to my interests.
